## مقدمه  
اگر دوست داری تجربهٔ کار تیمی با Git/GitHub را در یک پروژهٔ دانشگاهی یا متن‌باز تمرین کنی، این راهنما برای توست.

---

## A. فورک؛ نسخهٔ شخصی تو  
روی دکمهٔ **Fork** در صفحهٔ مخزن کلیک کن تا یک کپی از پروژه در حساب گیت‌هابت ساخته شود. این محیط شخصی بهترین جا برای آزمون و خطاست.
```bash
 https://github.com/artafps/SystemAlgoProcess 
```
---

## B. کلون به رایانهٔ خود  

```bash
git clone https://github.com/<YOUR_USERNAME>/<REPO_NAME>.git
cd <REPO_NAME>
```

---

## C. ایجاد شاخهٔ مجزا

برای هر ویژگی یا رفع باگ، یک شاخهٔ تازه بساز تا تغییراتت جدا بماند:

```bash
git checkout -b feat/improve-scheduler
```

---

## D. کدنویسی، تست و کامیت

1. تغییرات لازم را اعمال کن.
2. تست‌های پروژه را اجرا کن تا مطمئن شوی چیزی خراب نشده.
3. تغییرات را با پیام کامیت واضح ثبت کن:

```bash
git add .
git commit -m "docs: refine SRTF timeline example"
```

---

## E. پوش و Pull Request

```bash
git push origin feat/improve-scheduler
```

گیت‌هاب لینک **Compare & Pull Request** را نشان می‌دهد؛ کلیک کن و توضیحات مختصر اضافه کن.

---

## F. بازبینی و ادغام

* اعضای پروژه کد را بررسی می‌کنند و اگر لازم باشد بازخورد می‌دهند.
* پس از اعمال اصلاحات، زمانی که تغییرات بدون مشکل و مطابق استاندارد باشد، Pull Request ادغام می‌شود و نام تو در تاریخچهٔ پروژه ثبت خواهد شد.

---

## G نکته‌های پایانی

* قبل از شروع، **Issue**‌های باز را ببین تا کار تکراری انجام نشود.
* Lint و فرمت کد را رعایت کن تا بازبینی سریع‌تر پیش برود.
* اگر سؤالی داشتی، در قسمت Discussion یا گروه چت پروژه مطرح کن؛ جامعهٔ ما دوستانه است و خوشحال می‌شود کمک کند.

---
